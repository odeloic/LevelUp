# LevelUp — CLAUDE.md

Personal career roadmap tracker. Nuxt 4 + Drizzle ORM + Turso (libsql/SQLite).

---

## Stack

| Layer | Tech |
|---|---|
| Frontend | Nuxt 4 (`app/` directory layout), Vue 3, TypeScript |
| Server | Nuxt Nitro (H3) — `server/api/` routes |
| ORM | Drizzle ORM (`drizzle-orm/libsql`) |
| DB | Turso (libsql) in production, `file:./levelup.db` locally |
| Validation | Zod |

---

## NPM Scripts

```bash
npm run dev          # Nuxt dev server (hot reload)
npm run build        # Production build
npm run preview      # Preview production build
npm run db:seed      # Initialize DB from roadmap.sql (idempotent)
npm run db:introspect  # Regenerate server/db/migrations/schema.ts from DB
```

---

## Environment Variables


## Architecture

```
app/
  app.vue                    # Root Vue component

server/
  schemas/                   # Zod input schemas (shared by routes + queries)
    params.ts                # PhaseIdParam
    completions.ts           # LogCompletionBody
    user.ts                  # UpdateUserStateBody

  utils/
    response.ts              # ApiResponse<T>, success(), apiError()
    errors.ts                # ErrorCode, AppError, handleError()

  db/
    index.ts                 # Drizzle client (libsql)
    migrations/
      schema.ts              # Generated by drizzle-kit introspect
      relations.ts           # Generated relations
    queries/
      xp.ts                  # XP_VALUES, calcCompletionXp(), calcStreakBonus()
      user.ts                # getUserState(), updateUserState(), nextStreak()
      tasks.ts               # getTasksForPhase(), getTaskById()
      completions.ts         # logCompletion(), getEvidenceTrail()
      phases.ts              # getActivePhases(), getUnlockedPhases(), isPhaseUnlocked()

  api/
    phases/index.get.ts      # GET /api/phases (?all=true includes locked)
    phases/[id]/tasks.get.ts # GET /api/phases/:id/tasks
    completions/index.post.ts# POST /api/completions
    user/state.get.ts        # GET /api/user/state
    user/state.patch.ts      # PATCH /api/user/state

scripts/
  seed.ts                    # Runs roadmap.sql via client.executeMultiple()

roadmap.sql                  # Source of truth — schema + seed data
```

## API Response Shape

All routes return `ApiResponse<T>`:

```ts
{ ok: true;  data: T }
{ ok: false; error: { code: string; message: string; details?: unknown } }
```

Error codes: `PHASE_NOT_FOUND`, `TASK_NOT_FOUND`, `PHASE_LOCKED`, `INVALID_BODY`, `INVALID_PARAMS`, `USER_STATE_MISSING`, `INTERNAL`

## Route Handler Pattern

```ts
export default defineEventHandler(async (event) => {
  try {
    const body = Schema.parse(await readBody(event))
    return success(await queryFn(...))
  } catch (err) {
    handleError(err)  // always re-throws as H3 error
  }
})
```

## DB Access Patterns

```ts
// Single row (returns T | undefined)
const row = await db.select().from(table).where(eq(table.id, id)).get()

// Multiple rows
const rows = await db.select().from(table).where(eq(table.col, val)).orderBy(table.sortOrder)

// Insert + return
const [inserted] = await db.insert(table).values({...}).returning()

// Upsert (user_state auto-create)
await db.insert(userState).values(INITIAL).onConflictDoNothing()

// Always pass explicit completedAt — the schema default is a broken string literal
completedAt: new Date().toISOString()
```

## Phase Unlock Logic

A phase unlocks when `unlock_requires_phase_id IS NULL` OR all `is_deliverable=1` tasks in the required parent phase have at least one completion with `pr_url IS NOT NULL OR notes IS NOT NULL`.
